# 설치 검증 및 수정 기록

현재 환경: Python 3.12.3 / aarch64 (ARM64) / Linux

---

## [FIX-1] pyproject.toml — build-backend 경로 오류

**증상**: `pip install -e .` 실행 시 `BackendUnavailable: Cannot import 'setuptools.backends.legacy'`

**원인**: `setuptools.backends.legacy:build` 는 존재하지 않는 경로. 올바른 경로는 `setuptools.build_meta`

**수정**:
```toml
# 변경 전
build-backend = "setuptools.backends.legacy:build"

# 변경 후
build-backend = "setuptools.build_meta"
```

**파일**: `pyproject.toml`

---

## [FIX-2] pyproject.toml — setuptools 버전 상한 추가

**증상**: `torch 2.x requires setuptools<82, but you have setuptools 82.x`

**원인**: setuptools 82에서 torch와의 호환성 문제 발생

**수정**:
```toml
# 변경 전
requires = ["setuptools>=68"]

# 변경 후
requires = ["setuptools>=68,<82"]
```

**파일**: `pyproject.toml`, `setup_env.sh`

---

## [FIX-3] pybammsolvers cmake 빌드 실패 (aarch64)

**증상**: `pybammsolvers` wheel 빌드 시 cmake가 SuiteSparse/SUNDIALS 라이브러리를 찾지 못해 실패

**원인**:
- `pybammsolvers`는 IDAKLU solver (SuiteSparse + SUNDIALS) C++ 빌드 필요
- aarch64에 사전 빌드된 wheel 없음
- 시스템 라이브러리 설치에 sudo 필요

**해결 방법**: PyBaMM을 `--no-deps`로 먼저 설치 후, pybammsolvers 제외하고 나머지 의존성 수동 설치
- PyBaMM은 pybammsolvers 없이도 동작 (scipy/casadi solver 사용)
- IDAKLU solver는 대규모 DFN 시뮬레이션에만 성능 차이 유의미

**수정**: `pyproject.toml` pybamm 의존성에 no-binary 옵션 안내 추가, `setup_env.sh` 설치 전략 수정
