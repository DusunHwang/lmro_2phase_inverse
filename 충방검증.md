- 입력 데이터: `data/raw/toyo/sech2_pybamm_sample/Toyo_LMR_sech2_PyBaMM_0p1C_0p33C_0p5C_1C_2C.csv`
- 생성 모델: PyBaMM positive half-cell, `SPM`
- C-rate: `0.1C`, `0.33C`, `0.5C`, `1C`, `2C`
- 전압 cutoff: 충전 `4.65 V`, 방전 `2.5 V`
- Rest 조건: 충전/방전 사이 `10 min`, 방전/다음 충전 사이 `10 min`
- 출력 샘플링 period: `5 s`
- 총 데이터 포인트: `17264`
- 전체 전압 범위: `2.500010 V` to `4.649990 V`
- TOYO 파일 전류 부호: 충전 `+`, 방전 `-`
- 파서 입력 후 PyBaMM 내부 전류 부호: 충전 `-`, 방전 `+`

## dQ/dV Overlay

아래 그래프는 CC 충전/CC 방전 구간만 사용했다. 충전 곡선은 `+dQ/dV`, 방전 곡선은 `-dQ/dV`로 표시했다.

![C-rate별 충방전 dQ/dV overlay](data/raw/toyo/sech2_pybamm_sample/dqdv_overlay_by_crate.png)

## 계산 방법

- `dQ/dV`는 각 C-rate의 `CC-Chg`, `CC-Dchg` step에서 계산했다.
- Rest 구간은 제외했다.
- TOYO CSV의 `Capacity(mAh)`는 각 step 내 누적 용량이므로, step별로 `dQ/dV = gradient(Q) / gradient(V)`를 계산했다.
- 방전은 표시 규칙을 명확히 하기 위해 `abs(dQ/dV)`에 방전 부호 `-`를 부여했다.
- 노이즈성 미분 진동을 줄이기 위해 가능한 구간에는 Savitzky-Golay smoothing을 적용했다.

## 충전 dQ/dV 하한 전압이 C-rate별로 다른 이유

충전 곡선의 하한 전압이 `2.5 V`에서 시작하지 않는 것은 샘플링 부족 때문이 아니다. 각 cycle은 직전 방전에서 `2.5 V` cutoff에 도달한 뒤 10분 rest를 거치고, 이후 충전 전류가 걸린 terminal voltage로 `CC-Chg`가 시작된다. 이때 전류가 클수록 ohmic/kinetic overpotential이 커져 충전 첫 terminal voltage가 더 높게 나타난다.

5초 샘플링으로 재생성해도 충전 시작 전압은 `0.1C: 2.597222 V`, `0.33C: 2.659789 V`, `0.5C: 2.682792 V`, `1C: 2.723385 V`, `2C: 2.768708 V`로 유지된다. 반대로 방전은 모든 C-rate에서 `2.500010 V` cutoff까지 도달한다. 따라서 이 차이는 물리 모델의 전류 의존 terminal voltage shift이며, 데이터 포인트 부족으로 endpoint를 놓친 현상은 아니다.

### OCP / dQ/dV 함수 정의

각 phase OCP는 다음 형태로 정의했다.

```text
U(x) = b0 + b1*x + sum_i A_i*tanh((x - c_i)/w_i)
dU/dx = b1 + sum_i (A_i/w_i)*sech^2((x - c_i)/w_i)
```

이번 데이터에서는 같은 tanh/sech² basis를 유지하되, effective OCP가 `2.5 V` to `4.65 V`를 덮도록 phase OCP를 선형 스케일링했다. 저장된 basis grid는 `data/raw/toyo/sech2_pybamm_sample/ocp_dqdv_sech2_basis.csv`에 있다.

## 역추정 가능성 메모

현재 시뮬레이션은 기존 코드베이스의 `FallbackA` 구조를 사용한다. 이 구조에서는 phase별 OCP, diffusivity, particle radius가 PyBaMM에 직접 독립 phase로 들어가는 것이 아니라 다음 effective 값으로 들어간다.

```text
U_eff = frac_R3m * U_R3m + frac_C2m * U_C2m
D_eff = frac_R3m * D_R3m + frac_C2m * D_C2m
R_eff = frac_R3m * R_R3m + frac_C2m * R_C2m
```

따라서 이 synthetic data만으로 안정적으로 검증 가능한 것은 `U_eff`, `D_eff`, `R_eff`다. phase fraction과 phase별 OCP prior를 고정하거나 강하게 제한하면 phase별 `D/R` 산출 가능성을 확인할 수 있지만, 아무 제약 없이 데이터만 넣으면 phase별 `D_R3m`, `D_C2m`, `R_R3m`, `R_C2m`는 서로 상쇄되는 해가 생길 수 있다.
